# Key Reminders

- Investigate thoroughly before planning. Do NOT plan or write code without understanding the problem first.
- Plan your changes before implementing. Multi-agent reviews occur at plan and review stages.
- Keep your task notepad up to date as you work. Anything not written down can be lost to compaction.
- If you don't have a task notepad for the current work, create one from .claude/templates/notepad.md.
- Consult `.claude/rules/` before working in an area — rules are path-scoped and auto-loaded when you open matching files, but search by domain for planning.
- Update documentation after significant changes.
- Request code review after implementing (at least 1 native + 1 external subagent).
- Think iteratively: investigate → plan → implement → test → review.
